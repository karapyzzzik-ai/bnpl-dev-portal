# Delivery/Partial delivery

- The client's order installment is approved (status completed/successful order)
- The merchant delivers the item
- The merchant sends the "delivered" status via the API
- The system changes the status to "delivered"
- The system changes the status to "buyed"/"bought out"

**Method:** PUT

**URL**
- DEV: `https://dev.bnpl.kz/api/accounting/v1/changeStatus/json`
- STAGE: `https://stage.bnpl.kz/api/accounting/v1/changeStatus/json`

## Example request

```bash
curl -X 'PUT' \
  'https://dev.bnpl.kz/api/accounting/v1/changeStatus/json' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '[
  {
    "merchantId": "string",
    "orders": [
      {
        "amount": int,
        "orderId": "string",
        "status": "string"
      }
    ]
  }
]'
```

After sending the delivery request, the order status changes to "delivered". The "delivered" status is an intermediate system status and changes to "buyed" (the final successful order status) within 5-7 minutes.

In the `amount` field you can pass a value **less** than the original order amount, thereby triggering a "partial delivery". Example: a client placed an order for 100,000 tg. on installment, and upon delivery/pickup of the item is ready to take only 80,000 tg. worth.

Partial delivery condition: the order amount must be at least 500 tg.

## Successful response

```json
{
  "messageId": "",
  "SuccessfulResponses": [
    {
      "error": "false",
      "msg": "cтатус заказа по MerchantId = test.kz и MerchantOrderId = 000000 успешно обновлен",
      "merchantOrderId": "000000"
    }
  ],
  "ErrorResponses": []
}
```
