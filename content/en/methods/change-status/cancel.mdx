# Cancellation

**Condition:** the order must be in the "completed"/"Successful order" status

- The client's order installment is approved (status completed/successful order)
- The merchant sends the "canceled" status via the API
- The system changes the status to "canceled"/"cancelled"

**Method:** PUT

**URL**
- DEV: `https://dev.bnpl.kz/api/accounting/v1/changeStatus/json`
- STAGE: `https://stage.bnpl.kz/api/accounting/v1/changeStatus/json`

## Example request

```bash
curl -X 'PUT' \
  'https://dev.bnpl.kz/api/accounting/v1/changeStatus/json' \
  -H 'accept: application/json' \
  -H 'Content-Type: application/json' \
  -d '[
  {
    "merchantId": "string",
    "orders": [
      {
        "amount": 100000,
        "orderId": "string",
        "status": "string"
      }
    ]
  }
]'
```

## Successful response

```json
{
  "messageId": "",
  "SuccessfulResponses": [
    {
      "error": "false",
      "msg": "order status for MerchantId = test.kz and MerchantOrderId = 000000 successfully updated",
      "merchantOrderId": "000000"
    }
  ],
  "ErrorResponses": []
}
```
