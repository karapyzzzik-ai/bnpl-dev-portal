# Authorization

### 1. Authorization method for obtaining a Bearer token

Method: **POST**

##### Example request

```
curl --request POST \
  --url https://dev.bnpl.kz/api/users/api/v1/sign-in \
  --header 'Content-Type: application/json' \
  --header 'accept: application/json' \
  --data '{
"password": "password",
"username": "username"
}'
```

_If the integration is on the stage environment, the request should be sent to https://stage.bnpl.kz/api/users/api/v1/sign-in_

##### Request structure

| Parameter | Type | Required | Description | Validation rules |
| --- | --- | --- | --- | --- |
| username | string | yes | Access token | number of characters from 6 to 100. |
| password | string | yes | Refresh token | number of characters from 6 to 100. |

### 2. Method for refreshing the access token

Method: **POST**

##### Example request

```
curl -X 'POST' 'https://dev.bnpl.kz/api/users/api/v1/refresh' \
-H 'accept: application/json' \
-H 'Content-Type: application/json' \
-d '{
"refreshToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9.eyJVc2VySUQiOiI4NDhkODE0NC02ZjFlLTQ3NWMtODY3Mi05MzhlYzUyNmZlNWMiLCJleHAiOjE2NzgxODExMzF9.EA1sAShMgTBHtwwcIQAvEP-zNNhoYFutTYQyTD4OBng"
}'
```
