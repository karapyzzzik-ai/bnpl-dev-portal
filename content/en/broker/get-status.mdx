# Getting the final status

At the specified postlink in /preapp, the final result for the order will arrive

## Request

```json
{
    "id": "0db69d6c-d152-a54d-8d15-5617017e69cb",
    "billNumber": "",
    "status": "completed",
    "signature": "5af89c388202a2d8e28d6f626323c5d9dfd33c2b1cabc00bcfecf1d94a31f9a0"
}
```

### Request structure

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| id | uid | Yes | BNPL order identifier |
| billNumber | string | Yes | Merchant order number |
| status | string | Yes | Order completion status: completed |
| signature | string | Yes | Request signature |

Signature generation algorithm:

```
hmac(sha512('"status" + ":" + statusValue + ":" + "preappId" + ":" + preappIdValue + ":" + "billNumber" + ":" + billNumberValue', secretKey)
hmac(sha512("status:completed:preappId:7d301c9c-6291-4016-b951-cf58d6605685:billNumber:100", 'Partner.code'))
```

For their part, the broker hashes and verifies the signature's validity using the same principle

### Response

```json
{
    "response": "completed"
}
```
