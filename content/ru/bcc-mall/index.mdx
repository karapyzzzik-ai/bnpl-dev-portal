# Процесс подключение к онлайн торговому центру BCC Mall через Google Feed

1. Для тестирования корректной обработки фида необходимо предоставить прямой URL к нему.
2. Если партнер уже передает товары в Google Merchant Center, то нужно убедится, что передаются обязательные и дополнительные поля.

| | |
| --- | --- |
| Формат | Google RSS 2.0. ([Ссылка](https://support.google.com/merchants/answer/160589) на спецификацию от Google Merchant Center) |
| Размер | Не должен превышать - 80MB или 150 тысяч товаров |
| Аутентификация | Поддерживаются:<br />- HTTP Basic Auth<br />- API Key (статический токен) |

### Содержимое

Таблица с описанием тэгов фида, используемых в BCC Mall.

| Тэг | Краткое описание | Подробное описание и рекомендации |
| --- | --- | --- |
| **Обязательные** |  |  |
| g:id | Уникальный ID-товара. | Данный атрибут должен быть статичным в рамках обновления фида. **Важно:** если вы отправляете под ID товар "Adidas yeezy 500", то при обновлении фида данный ID не должен изменяться (то есть, нельзя отправлять под данным ID иной товар) |
| g:title | Название товара. | Рекомендация: в названии товара лучше указывать необходимые данные для клиента и без лишних данных (о доставке, о городе и т.д.) **Пример отправки заголовка:** "Yeezy 500 limited edition" |
| g:description | Описание товара. | **Рекомендация:** в описании товара лучше отправлять полное описание по товару, а характеристики товара лучше передавать в атрибуте `<g:product_detail>` |
| g:link | URL на страницу товара. | |
| g:image_link | URL на основную картинку товара. | Данный атрибут нужен для того, чтобы мы могли показать покупателю картинку товара. В случае если ссылка является нерабочей (404 ошибка), то в данном случае проставляется заглушка с текстом **(Фото временно недоступно)**. |
| g:availability | Наличие товара. | Данный атрибут необходим для понимания доступности товара. В случае если товара уже нет в наличии у мерчанта, то необходимо передавать значение "out_of_stock". (тогда в карточке товара появляется текст **"Товара нет в наличии"**) По данной причине необходимо держать feed-ы актуальными и чаще обновлять со стороны мерчанта. Доступные значения: `in_stock`, `out_of_stock` |
| g:price | Цена товара. | Изначальная цена товара. В случае наличия скидки, необходимо скидочную цену отправлять в атрибуте "sale_price". **Формат отправки цены:** 9 999.0 KZT (указание валюты необязательно) |
| g:size | Размер. | В данном атрибуте отправляется значение размера товара. **Рекомендация:** для удобства выбора покупателя, желательно отправлять размер в рамках СНГ значений. Где для обуви используются цифры (39, 40, 41 и т.д.), а для одежды буквы (S, M, L и т.д.). **Важно:** нельзя передавать в одном атрибуте несколько значений или диапазон размеров (к примеру "40-45", "40,41, 42, 43"); в рамках одного `<item>...</item>` передается только одно значение товара |
| g:size_system | Системы измерения (страна). | В данном атрибуте отправляется значение системы измерения размера. Доступные значения: RU, US, UK, EU, INT, DE, FR, JP, CN, IT, BR, MEX, AU |
| g:product_type | Категория. | В данном атрибуте отправляется "категория" и "подкатегория" товара на стороне мерчанта. В данном теге необходимо передавать полное дерево категорий. Разделителем может быть запятая или знак больше ("," или ">"). Пример: Одежда > Верхняя одежда > Куртки > Ветровки |
| g:color | Цвет. | В данном атрибуте необходимо передавать значение "цвета" товара. Рекомендация: отправлять значение цвета на "русском языке" |
| g:additional_image_link | Массив с дополнительными картинками товара. | В данный атрибут отправляются дополнительные фотографии товара. Пример отправки нескольких фотографий: `<g:additional_image_link>https://example.com/images/additional-image-1.jpg</g:additional_image_link>` `<g:additional_image_link>https://example.com/images/additional-image-2.jpg</g:additional_image_link>` По каждому товару необходимо не менее 3х дополнительных фото. |
| g:age_group | Возрастная группа. | В данном атрибуте отправляется возрастная категория товара. Возможные значения для передачи: newborn, infant, toddler, kids, adult |
| g:gender | Пол. | В данном атрибуте отправляется значение пола, к которому относится товар. Доступные значения: male, female, unisex |
| **Дополнительные** |  |  |
| g:item_group_id | ID группы товаров. | В данном атрибуте необходимо передавать ID, в рамках которого объединены 2 или более товаров. **Важно:** данный атрибут обязателен, если у товара имеется несколько размеров и цветов, для того чтобы покупатель смог выбрать нужный "размер" или "цвет". Пример: `<item>....<item_group_id>123</item_group_id>...</item>` `<item>...<item_group_id>123</item_group_id>...</item>` |
| g:brand | Бренд товара. | В данном атрибуте отправляется бренд товара. Наименование бренда на стороне BCC Mall выделяется в предпросмотре и в карточке товара. **Пример отправки бренда:** `<g:brand>Apple</g:brand>` |
| g:sale_price | Акционная/скидочная цена. | Акционная/скидочная цена, за которую сейчас мерчант продает товар. Данная цена не должна быть меньше или равна 0. |
| g:sale_price_effective_date | Период действия акционной цены. | Период времени, в течение которой действует акционная/скидочная цена. **Формат отправки даты:** 2023-12-31T23:59:00+0600 (ISO 8601) |
| g:expiration_date | Срок действия. | |
| g:availability_date | Дата начала. | |
| g:condition | Состояние товара | В данном атрибуте передается состояние товара. Возможные варианты в данном параметре: new (принимается только данное значение) |
| **Характеристика товара** |  |  |
| g:product_detail | Массив характеристик товара. | Поле details является массивом объектов. Объект имеет поля: section, name, value. В данном объекте передаются значения характеристики товара. (необязательно для заполнения) |
| g:section_name | Название группы | |
| g:attribute_name | Название атрибута | |
| g:attribute_value | Значение атрибута | |
| **Параметры товара (зависит от типа товара)** |  |  |
| g:energy_efficiency_class | Класс энергоэффективности. | |
| g:min_energy_efficiency_class | Класс минимальной энергоэффективности. | |
| g:max_energy_efficiency_class | Класс максимальной энергоэффективности. | |
| g:material | Материал товара. | В данном атрибуте отправляется значение материала, из которого сделан товар. |
| g:pattern | Рисунок или принт. | |
| g:size_type | Тип размера. | В данном атрибуте можно передавать тип размера для дополнительной информации покупателю. Доступные значения: regular, petite, maternity, big, tall, plus |
| g:product_length | Длина. | В данном атрибуте передается значение "длины" товара. Формат отправки: `{length} {unit}`. Доступные значения для "unit": cm, in. **Пример:** `<g:product_length>30 cm</g:product_length>` |
| g:product_width | Ширина. | В данном атрибуте передается значение "ширины" товара. Формат отправки: `{width} {unit}`. Доступные значения для "unit": cm, in. **Пример:** `<g:product_width>30 cm</g:product_width>` |
| g:product_height | Высота. | В данном атрибуте передается значение "высоты" товара. Формат отправки: `{height} {unit}`. Доступные значения для "unit": cm, in. **Пример:** `<g:product_height>5 kg</g:product_height>` |
| g:product_weight | Вес. | В данном атрибуте передается значение "веса" товара. Формат отправки: `{weight} {unit}`. Доступные значения для "unit": kg, lb. **Пример:** `<g:product_weight>5 kg</g:product_weight>` |
| g:product_highlight | Краткое описание ключевых характеристик. | В данном атрибуте передается информация по ключевым кратким характеристикам товара. Пример: `<g:product_highlight>Короткое описание ключевых характеристик</g:product_highlight>` |
| **Информация по точкам самовывоза** |  |  |
| bnpl:pickup | Массив по адресам филиалов, в которых имеется товар | Необходимо передавать в рамках атрибута `<bnpl:pickup>...</bnpl:pickup>` данные по одному адресу |
| bnpl:id | ID | Уникальный ID точки самовывоза. Не должна повторяться с другими точками. Обязательное поле. |
| bnpl:region | Область | Область города. В случае, если город республиканского значения, значение может быть пустым. Условно-обязательное поле. |
| bnpl:city | Город | Точка в городе, из которого можно забрать товар. Обязательное поле. |
| bnpl:street | Улица | Улица точки в городе, из которого можно забрать товар. Обязательное поле. |
| bnpl:building | Номер здания | Номер здания в адресе. Обязательное поле. |
| bnpl:apartment | Номер офиса | Номер офиса, если есть. Необязательное поле. |
| bnpl:available | Признак наличия | Признак наличия товара в этой точке для самовывоза. Обязательное поле. Доступные значения: yes, no |
| **Информация по новой Коллекции** |  |  |
| bnpl:collection | Наименование коллекции | Передается информация по наименованию коллекции. Если в одной (новой) коллекции имеется, к примеру, 100 товаров, то в рамках этих 100 товаров `<item>....</item>` должны передаваться наименование коллекции (метка, по которой мы понимаем, что товары из одной коллекции) |

### Пример оформления фида

```xml
<?xml version="1.0"?>
<rss version="2.0" xmlns:g="http://base.google.com/ns/1.0">
    <channel>
        <item>
            <!--Обязательные-->
            <g:id>100555</g:id>
            <g:title>Название товара</g:title>
            <g:description>
                <![CDATA[
                    Это описание товара.
                ]]>
            </g:description>
            <g:link>https://example.com/link-to-product-page</g:link>
            <g:image_link>https://example.com/images/main-image.jpg</g:image_link>
            <g:availability>In stock</g:availability>
            <g:price>9 999.0 KZT</g:price>

            <!--Дополнительные-->
            <g:product_type>Книги > Изучение языков > Какая-то литература</g:product_type>
            <g:brand>Apple</g:brand>
            <g:sale_price>9 899.0 KZT</g:sale_price>
            <g:sale_price_effective_date>2023-09-25T00:00:00+0600/2023-09-30T23:59:00+0600</g:sale_price_effective_date>
            <g:condition>new</g:condition>
            <g:expiration_date>2023-12-31T23:59:00+0600</g:expiration_date>
            <g:additional_image_link>https://example.com/images/additional-image-1.jpg</g:additional_image_link>
            <g:additional_image_link>https://example.com/images/additional-image-2.jpg</g:additional_image_link>
            <g:availability_date>2023-09-01T00:00:00+0600</g:availability_date>

            <!--Характеристики товара-->
            <g:product_detail>
                <g:section_name>Название группы</g:section_name>
                <g:attribute_name>Артикул товара</g:attribute_name>
                <g:attribute_value>1000</g:attribute_value>
            </g:product_detail>
            <g:product_detail>
                <g:attribute_name>Жанр</g:attribute_name>
                <g:attribute_value>Какая-то литература</g:attribute_value>
            </g:product_detail>

            <!--Параметры товара (зависит от типа товара) -->
            <g:item_group_id>100554</g:item_group_id>
            <g:adult>no</g:adult>
            <g:energy_efficiency_class>A++</g:energy_efficiency_class>
            <g:min_energy_efficiency_class>A</g:min_energy_efficiency_class>
            <g:max_energy_efficiency_class>A+++</g:max_energy_efficiency_class>
            <g:age_group>kids</g:age_group>
            <g:color>White</g:color>
            <g:gender>male</g:gender>
            <g:material>cotton/polyester/elastane</g:material>
            <g:pattern>striped</g:pattern>
            <g:size>M</g:size>
            <g:size_type>regular</g:size_type>
            <g:size_system>EU</g:size_system>
            <g:product_length>30 cm</g:product_length>
            <g:product_width>30 cm</g:product_width>
            <g:product_height>170 cm</g:product_height>
            <g:product_weight>5 kg</g:product_weight>
            <g:product_highlight>Короткое описание ключевых характеристик</g:product_highlight>

          <!--Точки самовывоза товара-->
            <bnpl:pickup>
                <bnpl:id>1</bnpl:id>
                <bnpl:region>Акмолинская область</bnpl:region>
                <bnpl:city>Кокшетау</bnpl:city>
                <bnpl:street>Абая 1</bnpl:street>
                <bnpl:building>100</bnpl:building>
                <bnpl:apartment>1</bnpl:apartment>
                <bnpl:available>yes</bnpl:available>
            </bnpl:pickup>

            <bnpl:pickup>
                <bnpl:id>2</bnpl:id>
                <bnpl:region></bnpl:region>
                <bnpl:city>Алматы</bnpl:city>
                <bnpl:street>Абая 1</bnpl:street>
                <bnpl:building>100</bnpl:building>
                <bnpl:apartment></bnpl:apartment>
                <bnpl:available>no</bnpl:available>
            </bnpl:pickup>

            <bnpl:collection>Осень-зима 2024</bnpl:collection>
        </item>
    </channel>
</rss>
```

### Рекомендации по оформлению фида

Для корректного деления товаров в BCC Mall рекомендуем добавить дополнительную информацию в ваш товарный фид, которая потребуется для идентификации и обработки заказа менеджерами (артикул товара и другие доп. атрибуты).

Например, возьмем два одинаковых товара с отличающимся объемом: один 50 мл, другой 100 мл, тогда оформление фида будет выглядеть следующим образом:

```xml
<!--Указание объема:-->
<g:product_detail>
    <g:section_name>Название группы атрибута</g:section_name>
    <g:attribute_name>Объем</g:attribute_name>
    <g:attribute_value>50 мл</g:attribute_value>
</g:product_detail>

<!--Указание артикула товара:-->
<g:product_detail>
    <g:section_name>Название группы атрибута</g:section_name>
    <g:attribute_name>Артикул товара</g:attribute_name>
    <g:attribute_value>8411061059548</g:attribute_value>
</g:product_detail>
```

Аналогично и для другого варианта этого товара.

Таким же образом можно передавать другие необходимые атрибуты товара.
