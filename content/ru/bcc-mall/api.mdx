# API для получения заказов из BCC Mall

Данный API предназначен для отправки успешных/одобренных заказов с BCC Mall ([https://bnpl.kz/customer-cabinet](https://bnpl.kz/customer-cabinet)) заказов в систему партнера (админка сайта/CRM/1C).

### Процесс получения и обработки заказа из BCC Mall

1. Клиент оформляет заказ в BCC Mall и получает одобрение
2. BNPL передает необходимый набор данных на endpoint партнера
3. Партнер обрабатывает полученные данные и просаживает заказ в свою систему
4. После обработки заказа дальнейшее изменение статусов заказа менеджер производит через портал мерчанта или свою систему (при полной интеграции с bnpl)

Подробнее о смене статусов [тут](/ru/methods/change-status/delivery).

### Реализация API

На стороне BNPL имеется сервис, который формирует скоп данных в JSON и отправляет на end-point партнера.

Данные, передающиеся POST запросом в формате JSON:

##### Описание атрибутов

**Реализация API на стороне партнера:** партнеру необходимо на своей стороне реализовать end-point, куда BNPL сможет отправлять данные по заказу. После получения JSON партнер самостоятельно парсит данные и формирует активный заказ в своей учетной системе.

- HTTP код ответа на запрос должен быть успешным 2XX.
- В случае ошибок наша система попробует несколько раз повторить запрос. В связи с этим могут прийти повторные запросы с одинаковыми данными, ваша система должна уметь обрабатывать повторные запросы.

**При получении заказа с BCC Mall товар автоматически должен переходить из свободного остатка в резерв.**

**В случае наличия авторизации** для доступа к вашему эндпоинту необходимо предоставить следующие данные:

- название заголовка токена и само значение токена авторизации в случае статичного токена авторизации (static token auth);
- логин и пароль пользователя в случае базовой авторизации (basic auth).

Пример необходимых для тестирования данных со стороны партнера:

_Ссылка на API: https://testsite/local/integrations/bnpl.php_

_Для тестирования авторизации токен установлен таким вот способом:_

_Header("Authorization: Bearer 1234567890")_

_Тестовый bearer-токен - 1234567890_
