# Лимит на рассрочку на сайте партнеров

**Лимит на рассрочку на сайте партнеров** – специальный функционал партнерского сервиса BNPL/Рассрочка 0-0-4, позволяющий клиенту узнать доступный ему лимит до фактического оформления заказа.

_Клиент, получивший лимит, может оформлять заказ в рамках одобренной суммы на сайте партнера, без опасения получить отказ._

### Флоу лимита на рассрочку

1. Экран онбординга. Ввод ИИНа и номера телефона

**_ИИН и номер телефона должны быть связаны в БМГ (egov.kz)_**

![Ввод смс-кода](https://bnpl.kz/dev-portal/_next/static/media/3.eef1fe02.jpg)

### Подключение

1. Разместить маркетинговые элементы на страницах сайта партнера

- Хедер
- Карточки товаров
- Модальные окна

![Скоринг](https://bnpl.kz/dev-portal/_next/static/media/7.68503b11.jpg)
![Успешное оформление рассрочки](https://bnpl.kz/dev-portal/_next/static/media/8.f583e5ac.jpg)

2. На всех элементах разместить ссылку `http://dev.bnpl.kz/qr/scoring?partnerCode=bnpl.kz&pointCode=01-0010-L1&site=сайт_партнера` с открытием в новой вкладке

В параметре `site` указать ссылку на страницу, куда вернуть клиента после завершения процесса по лимиту, или ссылку на главную страницу сайта.

3. Настроить редирект клиента на ту же страницу, с которой он перешел на флоу лимита на рассрочку

Для того, чтобы по окончанию флоу лимита на рассрочку вернуть клиента на ту же страницу, с которой он перешел, необходимо в параметр `site=` динамически передавать ссылку на текущую страницу.

Это можно сделать следующими способами:

##### 3.1. Через бэкэнд-шаблонизатор

Если ссылка формируется на серверной стороне с использованием шаблонизатора (например, Twig, Blade, Smarty и др.), вы можете передавать текущий URL страницы в параметре `site`.

**Пример:**

```html
<a href="https://bnpl.kz/qr/scoring?partnerCode=bnpl.kz&pointCode=01-0010-L1&site=' . urlencode($current_url) . '">Перейти</a>
```

- **partnerCode:** ваш код партнёра.
- **current_url:** переменная, содержащая URL текущей страницы.
- Убедитесь, что URL корректно закодирован для передачи в параметре (например, с использованием функций `urlencode()` или их аналогов).

##### 3.2. Через JavaScript на фронте

Если ссылка не может быть сформирована на сервере, вы можете динамически добавлять текущий URL в параметр `site` с помощью JavaScript.

**Пример:**

```html
<a id="dynamic-link" href="#">Перейти</a>
<script>
document.addEventListener("DOMContentLoaded", function () {
    const link = document.getElementById("dynamic-link");
    if (link) {
      const currentUrl = encodeURIComponent(window.location.href);
      link.href = "https://bnpl.kz/qr/scoring?partnerCode=bnpl.kz&pointCode=01-0010-L1&site=${currentUrl}";
}});
</script>
```

- **window.location.href:** получает полный URL текущей страницы.
- **encodeURIComponent:** кодирует URL, чтобы он был корректно передан в параметре.

##### 3.3. Через обработчик клика на ссылке

Если нужно добавлять параметр `site` только в момент клика, можно использовать обработчик событий.

**Пример:**

```html
<a id="dynamic-link" href="https://bnpl.kz/qr/scoring?partnerCode=bnpl.kz&pointCode=01-0010-L1">Перейти</a>
<script>
    const link = document.getElementById("dynamic-link");

    link.addEventListener("click", function (event) {
        // предотвращаем переход по ссылке до изменения href
        event.preventDefault();

        // текущий URL
        const currentUrl = encodeURIComponent(window.location.href);

        // обновленная ссылка
        const newLink = `${link.href}&site=${currentUrl}`;

        // перенаправляем на обновленную ссылку
        window.location.href = newLink;
});
</script>
```

- Обработчик позволяет избежать передачи лишних данных, если пользователь не кликает по ссылке.
- Важно: обработчик должен быть подключен только на нужных страницах.

### Рекомендации

1. **Кодирование URL:** обязательно используйте `encodeURIComponent` для корректной передачи ссылок.
2. **Минимизация кода:** используйте бэкэнд-шаблонизатор, если это возможно, чтобы уменьшить количество клиентских скриптов.
3. **Тестирование:** убедитесь, что ссылки корректно формируются и переходы работают на всех страницах.
